Application documentation

CRM Sports - API Documentation

Setup Instructions

Prerequisites

  • Node.js (v16 or higher)
  • PostgreSQL (v14 or higher)
  • npm or yarn

Installation

  1. Clone the repository
  2. Install dependencies:
Example :
npm install

Authentication

All endpoints require authentication via JWT token in the Authorization header: Bearer <token>

Endpoints

Customers

  • GET /api/customers

    • List all customers
    • Auth required
    • Query params: none

  • GET /api/customers/:id

    • Get customer details
    • Auth required
    • Params: id (string)

  • POST /api/customers

    • Create new customer
    • Auth required
    • Body:Example :
      {
  "name": "string",
  "email": "string",
  "phone": "string",
  "company": "string"
}

  • PUT /api/customers/:id

    • Update customer
    • Auth required
    • Params: id (string)
    • Body: same as POST

  • DELETE /api/customers/:id

    • Delete customer (soft delete)
    • Admin only
    • Params: id (string)

Orders

  • GET /api/orders

    • List all orders
    • Auth required
    • Query params:
      • status (optional): "pending" | "completed" | "cancelled"

  • GET /api/orders/:id

    • Get order details
    • Auth required
    • Params: id (string)

  • GET /api/orders/customer/:customerId

    • Get customer's orders
    • Auth required
    • Params: customerId (string)

  • POST /api/orders

    • Create new order
    • Auth required
    • Body:Example :
      {
  "customerId": "string",
  "products": [
    {
      "productId": "string",
      "quantity": number
    }
  ],
  "status": "pending"
}

  • PUT /api/orders/:id/status

    • Update order status
    • Admin only
    • Params: id (string)
    • Body:Example :
      {
  "status": "completed" | "cancelled"
}

Products

  • GET /api/products

    • List all products
    • Auth required
    • Query params:
      • category (optional): string
      • inStock (optional): boolean

  • GET /api/products/:id

    • Get product details
    • Auth required
    • Params: id (string)

  • POST /api/products

    • Create new product
    • Admin only
    • Body:Example :
      {
  "name": "string",
  "description": "string",
  "price": number,
  "category": "string",
  "stock": number
}

  • PUT /api/products/:id

    • Update product
    • Admin only
    • Params: id (string)
    • Body: same as POST

  • DELETE /api/products/:id

    • Delete product (soft delete)
    • Admin only
    • Params: id (string)

Configuration

  • GET /api/config

    • Get all configurations
    • Auth required

  • POST /api/config

    • Create configuration
    • Admin only
    • Body:Example :
      {
  "key": "string",
  "value": "string"
}

  • PUT /api/config/:id

    • Update configuration
    • Admin only
    • Params: id (string)
    • Body: same as POST

  • DELETE /api/config/:id

    • Delete configuration
    • Admin only
    • Params: id (string)

Dashboard

  • GET /api/dashboard/stats

    • Get overall statistics
    • Auth required
    • Returns: counts of customers, orders, products

  • GET /api/dashboard/summary

    • Get orders summary by status
    • Auth required
    • Returns: grouped orders by status

  • GET /api/dashboard/sales

    • Get detailed sales statistics
    • Admin only
    • Returns: completed orders with products

Tasks

  • GET /api/tasks

    • List all tasks
    • Auth required
    • Query params:
      • status: "pending" | "completed"
      • priority: "high" | "medium" | "low"

  • POST /api/tasks

    • Create new task
    • Auth required
    • Body:Example :
      {
  "title": "string",
  "description": "string",
  "dueDate": "date",
  "priority": "high" | "medium" | "low",
  "assignedTo": "userId"
}

Appointments

  • GET /api/appointments

    • List all appointments
    • Auth required
    • Query params:
      • date: "YYYY-MM-DD"
      • status: "scheduled" | "completed" | "cancelled"

  • POST /api/appointments

    • Create appointment
    • Auth required
    • Body:Example :
      {
  "clientId": "string",
  "date": "datetime",
  "duration": number,
  "notes": "string"
}

Authentication

  • POST /api/auth/login

    • User login
    • Body:Example :
      {
  "email": "string",
  "password": "string"
}
    • Returns: JWT token

  • POST /api/auth/register

    • Create new user (Admin only)
    • Body:Example :
      {
  "name": "string",
  "email": "string",
  "password": "string",
  "role": "user" | "admin"
}

Reports

  • GET /api/reports/sales

    • Get sales report
    • Admin only
    • Query params:
      • startDate: "YYYY-MM-DD"
      • endDate: "YYYY-MM-DD"

  • GET /api/reports/customers

    • Get customer activity report
    • Admin only
    • Query params:
      • period: "daily" | "weekly" | "monthly"

Notifications

  • GET /api/notifications

    • Get user notifications
    • Auth required
    • Query params:
      • read: boolean

  • PUT /api/notifications/:id/read

    • Mark notification as read
    • Auth required

Error Responses

All endpoints return standard HTTP status codes:

  • 200: Success
  • 201: Created
  • 400: Bad Request
  • 401: Unauthorized
  • 403: Forbidden
  • 404: Not Found
  • 500: Server Error

Error response body:

Example :
{
  "error": "Error message"
}
